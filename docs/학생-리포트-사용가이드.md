# 학생 리포트 기능 사용 가이드

이 문서는 **학생 상세** 화면을 중심으로 **월간·분기·반기·연간** 리포트를 작성하고, 저장된 내용을 다시 열어 보거나 수정하는 방법을 정리합니다.

---

## 1. 화면 이동 경로

| 목적           | 경로 |
|----------------|------|
| 학생 목록      | `/students` |
| 학생 상세      | `/students/:학생ID` |
| 월간 리포트 작성·보기 | `/students/:학생ID/monthly/new` (쿼리로 `ym`, `step` 지정) |
| 분기·반기·연간 리포트 | `/students/:학생ID/period/new` (쿼리로 `type`, `end_ym`, 필요 시 `q_id` / `h_id` / `y_id`) |
| **한우리 도서관** (도서 코너) | `/books` — 상단 네비게이션의 **「도서」**와 동일 |

로그인 후 보호된 메뉴에서 위 주소로 이동합니다.

### 1.1 상단 네비게이션 — 도서 (한우리 도서관)

- 화면 상단에는 **학생**과 **「도서」** 링크가 있습니다. 넓은 화면에서는 가로 메뉴, 좁은 화면에서는 **☰ 메뉴**를 열면 같은 링크가 보입니다.
- **「도서」**를 누르면 **`/books`** 로 이동하며, 페이지 제목은 **「한우리 도서관」**입니다.
- 여기서는 월간 리포트와 별도로 **도서명·출판사·저자(또는 역자)** 를 넣고 **YES24**에서 메타데이터를 가져온 뒤, **도서관에 저장**해 두는 흐름을 쓸 수 있습니다. 미리 등록해 두면, 월간 **3단계「이달의 도서」**에서 **도서함(검색)** 으로 같은 책을 빨리 찾을 수 있습니다.

---

## 2. 학생 상세 (`/students/:id`)

### 2.1 회차별 월간 리포트 표

- **회차**: 학생 **등록월**을 1회차로 두고, 한 달씩 번호가 올라갑니다.
- **AI 레포트 생성** 버튼: 해당 회차(해당 달)의 **새 월간·분기·반기·연간** 작성 화면으로 갈 수 있는 메뉴가 열립니다.  
  - 3의 배수 회차에만 **분기** 링크가 활성화되고, 6의 배수에 **반기**, 12의 배수에 **연간**이 활성화됩니다.
- **회차 아래 `YYYY-MM`**: 그 회차가 가리키는 실제 연·월입니다.
- **신호등(원형 표시)**: 그 달까지 **의무로 요구되는** 월간·분기·반기·연간이 모두 채워졌는지 요약합니다. (범례는 표 아래에 설명되어 있습니다.)

### 2.2 회차별 5대 역량 점수 차트

- 월간 리포트에 저장된 **역량 점수**가 있으면 꺾은선으로 표시됩니다.
- **X축의 `N회` (파란 밑줄)**: 저장된 레포트가 하나라도 있으면 클릭할 수 있습니다.
- 클릭 시 **레포트 보기** 메뉴가 열리며, **실제로 저장된 항목만** 링크로 나타납니다.
  - **월간 레포트 보기**: 해당 회차 달의 월간 저장본 (`step=6` 확인 화면과 동일한 흐름으로 수정 가능).
  - **분기별 레포트 보기**: 해당 회차 달이 속한 **달력 분기**에 저장된 분기 리포트가 있으면 표시됩니다. (DB에 분기 마지막 달이 아닌 다른 달로 저장된 경우에도 같은 분기면 매칭됩니다.)
  - **반기·연간**: 해당 회차에 맞는 기간에 저장본이 있을 때만 표시됩니다.

> 차트에 점이 거의 없을 때는, “월간 역량 점수가 없다”는 안내와 함께 X축 클릭으로 저장본 메뉴를 쓸 수 있다는 설명이 나올 수 있습니다.

---

## 3. 월간 리포트 (`/students/:id/monthly/new`)

### 3.0 마법사 단계 순서 (전체 흐름)

월간 리포트는 아래 순서로 진행됩니다. 상단 **단계 탭**으로 이동할 수 있는지는, 해당 달에 **저장된 리포트가 있는지**에 따라 달라집니다(저장 후에는 탭 이동이 가능).

| 단계 | 제목 | 하는 일(요약) |
|:----:|------|----------------|
| 1 | 이달의 성장 모먼트 | 성장 모먼트 입력 |
| 2 | 이달의 글쓰기 | 글쓰기 이미지 첨부(최대 2장) |
| **3** | **이달의 도서** | **도서함 검색 → (필요 시) YES24 등록 → 검색 결과에서 선택** |
| 4 | 5대 역량 | 역량별 코멘트 |
| 5 | 따뜻한 한마디 · 생성 | 초안·AI 생성 등 |
| 6 | 결과 확인 · 저장 | 최종 화면에서 수정 후 저장 |

### 3.1 새로 작성

1. 학생 상세에서 **AI 레포트 생성** → **월간 레포트 생성하기**, 또는 주소에 `?ym=YYYY-MM`을 붙여 해당 달로 진입합니다.
2. **3.0**의 순서대로 단계를 진행합니다. **3단계**에서 도서를 고르는 방법은 **3.3**을 참고하세요.
3. 마지막 **6. 결과 확인 · 저장**에서 내용을 다듬고 저장합니다.

### 3.2 저장된 월간 열기·수정

- 주소 예:  
  ` /students/:id/monthly/new?ym=2025-03&step=6 `
- **`step=6`**: 저장된 해당 달 리포트를 불러와 **확인 화면과 동일한 UI**에서 섹션별로 수정할 수 있습니다.
- **학생 상세**의 X축 **월간 레포트 보기** 링크도 같은 형태로 연결됩니다.

### 3.3 이달의 도서 (3단계): 도서함 검색 · YES24 크롤링 · 선택

**목표:** 리포트에 넣을 책을 **최대 2권**까지 고릅니다. 먼저 **이미 도서관(DB)에 있는지** 찾고, 없으면 **YES24**에서 정보를 가져와 도서함에 넣은 뒤 선택합니다.

#### (1) 도서명으로 도서함 검색

1. **도서명**(필수)을 입력합니다.
2. **「검색」**을 누르면 Supabase 등에 연결된 **도서함**에서 **제목 부분 일치** 검색이 실행됩니다.
3. 결과 목록이 나오면, 책 카드를 눌러 **선택된 도서** 칸에 넣습니다. (이미 고른 책은 유지되며, 도서명만 바꿔 다시 검색할 수 있습니다.)

#### (2) 검색 결과가 없을 때 — YES24에서 도서 등록 (크롤링)

1. **「검색」** 후 목록이 **비어 있으면**, 안내 문구에 따라 **출판사**, **저자 또는 역자**를 입력합니다. (YES24에 맞게 적는 것이 검색 성공에 유리합니다.)
2. **「YES24에서 도서 등록」**을 누릅니다.
3. 내부적으로는 **YES24 검색 API**(로컬 개발 시에는 Vite 개발 서버의 플러그인, 배포 시에는 Cloud Run 등에 설정한 URL·API 키)를 호출해 표지·소개·분류·AI 키워드 등을 채웁니다. 진행 상황은 **한우리 도서 검색 진행** 영역(로그 메시지)에 쌓입니다.
4. 동일 제목이 이미 도서함에 있으면 크롤링 대신 **기존 행**을 불러와 목록에 보여 줍니다.
5. 성공 시 **도서함(upsert)** 에 저장되고, 검색 결과 맨 앞에 새 행이 붙습니다. DB에 `book_id`가 붙으면 **분류·키워드**를 DB에서 한 번 더 맞춰 읽어 옵니다.

> **YES24 버튼이 안 보이거나** “이 사이트에서는 YES24 자동 등록을 사용할 수 없습니다”가 나오는 경우: **개발 모드(`import.meta.env.DEV`)** 에서는 기본적으로 사용 가능하고, **프로덕션**에서는 Cloud Run 베이스 URL과 API 키가 모두 설정되어 있어야 합니다. (`.env`의 `VITE_YES24_API_URL`, `VITE_YES24_API_KEY` 및 `src/config/yes24CloudRun.ts` 등 프로젝트 설정 참고)

#### (3) 선택 완료 후

- **선택된 도서**에 1권 이상 있어야 **다음** 단계(4. 5대 역량)로 갈 수 있습니다.
- 최종 저장 시 선택 도서는 월간 리포트의 **도서 키워드·표지 메타**와 연결되며, 이후 **분기 리포트**의 도서·마인드맵 단계에서도 참조될 수 있습니다.

#### 네비의 한우리 도서관과 같이 쓰기

- **상단 「도서」→ 한우리 도서관**에서 미리 **YES24 검색 후 도서관에 저장**해 두면, 월간 3단계의 **「검색」**만으로 같은 책을 찾아 선택하기 쉬워집니다.

## 4. 분기 리포트 (`/students/:id/period/new`)

### 4.1 쿼리 파라미터

| 파라미터 | 설명 |
|----------|------|
| `type`   | 기간 종류. 분기는 **`3m`** (기본값이 3m인 경우가 많음). |
| `end_ym` | 분기 구간의 기준 **마지막 달** `YYYY-MM`. 학생 상세의 “분기 작성” 링크에 포함됩니다. |
| `q_id`   | (선택) 이미 저장된 분기 리포트의 **ID**. 있으면 **그 저장본을 불러와** 작성 화면과 **동일한 단계·동일한 최종 레이아웃**에서 보고 수정합니다. |

### 4.2 새로 작성하는 흐름

1. 학생 상세에서 **AI 레포트 생성** → **분기별 레포트 생성하기** 등으로 `type=3m&end_ym=...` 링크로 들어갑니다.
2. 단계 탭: **1. best 글쓰기 선정** → **2. 지식 마인드맵** → **3. 성장 인사이트** → **4. 선생님의 따뜻한 한마디** → **5. 레포트 확인 · 저장**
3. 4단계에서 **레포트 생성하기**(AI)를 실행하면 5단계로 이동하며, 5단계에서 블록별 **수정** 후 **분기 리포트 저장**으로 Supabase에 반영합니다.

### 4.3 저장된 분기 열기·수정 (X축 “분기별 레포트 보기” 등)

- 주소 예:  
  ` /students/:id/period/new?type=3m&end_ym=2025-06&q_id=... `
- `end_ym`이 URL에 없어도, `q_id`만으로 서버에서 분기 행을 찾으면 **그 행의 `quarter_end_ym`**으로 구간이 잡힙니다.
- **저장본 보기**로 들어와도 **단계 탭 전체**를 쓸 수 있고, 기본으로 **5단계(레포트 확인 · 저장)** 와 작성 직후와 **같은 스타일의 결과 UI**가 열립니다. 내용을 고친 뒤 다시 **분기 리포트 저장**하면 업데이트됩니다.
- 작성 중에는 일정 시간마다 **초안 동기화**(Supabase 설정 시)가 동작할 수 있습니다.

## 5. 반기·연간 리포트 (저장본 보기)

- **반기**: `type=6m`, `end_ym`, 필요 시 `h_id`  
- **연간**: `type=12m`, `end_ym`, 필요 시 `y_id`

학생 상세의 **AI 레포트 생성** 메뉴에서 신규 작성 링크로 들어갈 수 있습니다.

현재 앱 동작 기준:

- **분기(`q_id`)**: 위 4절처럼 **작성 마법사와 동일한 화면**에서 보기·수정이 가능합니다.
- **반기(`h_id`)·연간(`y_id`)**: 저장본만 열 때는 **요약 전용(읽기 중심) 화면**으로 열릴 수 있습니다. 신규 작성·대규모 UI는 학생 상세에서 안내되는 링크를 따르세요.

(이후 반기·연간도 분기와 같이 통합 편집 UI로 맞출 경우, 이 절만 업데이트하면 됩니다.)

## 6. 자주 묻는 상황

**Q. 3회차를 눌렀는데 분기 “보기”가 안 보였다.**  
→ 해당 회차 달이 속한 **달력 분기**에 `q_reports` 저장이 있어야 합니다. 과거에는 DB `quarter_end_ym`과 “분기 마지막 달”만 일치할 때만 메뉴에 나왔으나, 지금은 **같은 분기면** 찾아서 메뉴에 표시합니다.

**Q. 분기를 열었는데 예전처럼 텍스트만 있는 빈 화면이다.**  
→ 최신 버전에서는 `q_id`로 들어올 때 **작성 화면과 같은 마법사·5단계 뷰**로 열립니다. 브라우저 캐시·배포 버전을 확인해 주세요.

**Q. 월간 `step=6`이 뭔가요?**  
→ 월간 리포트 마법사의 **마지막 확인 단계** 번호로, 저장분을 불러와 그 화면에서 수정하는 데 사용됩니다.

**Q. 월간에서 YES24가 안 된다.**  
→ **3.3**의 프로덕션 조건(Cloud Run URL·API 키)과 개발/배포 환경을 확인하세요. 한우리 도서관(`/books`)의 **「도서 검색 (YES24)」**도 같은 조건입니다.

---

## 7. 데이터·환경

- 학생·월간·분기 등 **저장 데이터**는 Supabase 등 프로젝트에 연결된 백엔드에 따라 달라집니다. 로컬/배포 환경의 `.env` 및 Supabase 설정을 맞춰 주세요.
- **기간 리포트 목록**(분기·반기·연간)을 불러오려면 Supabase가 구성되어 있어야 합니다.
- **YES24 연동**: 월간 3단계·한우리 도서관(`/books`) 공통. 배포 시 `VITE_YES24_API_URL`, `VITE_YES24_API_KEY`(및 코드의 Cloud Run 설정)를 확인하세요.

---

## 8. 변경 이력 요약 (기능 관점)

- 학생 상세 **차트 X축**: 저장분이 있는 경우 **월간·분기·반기·연간 보기** 메뉴.
- 분기 저장본 **`q_id` 진입**: 읽기 전용 요약 대신 **작성과 동일한 단계형 편집·결과 UI**.
- 상세 화면 하단의 **통합 “저장된 레포트 목록”** 블록은 제거되었고, 열람은 **차트 X축 메뉴**와 각 작성 페이지 위주로 진행합니다.

문의나 개선 제안은 저장소 이슈/담당 채널로 남겨 주시면 됩니다.
